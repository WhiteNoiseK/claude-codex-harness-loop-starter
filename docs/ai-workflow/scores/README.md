This directory holds the 6-stage (RED / GREEN / VERIFY / REVIEW / FIX / AUDIT) scores
and hallucination-audit results as per-task JSON.

## File Naming Convention

```text
<task_id>.json          latest integrated record (use this format for new tasks)
<task_id>.verify.json   Stage 3 VERIFY standalone artifact (generated by harness_run_verify.sh)
EXAMPLE_*.json          teaching example (not a real task — never use as gate input)
```

> The task_id syntax follows `.harness.toml [task_id].regex` (multi-segment allowed — e.g., `{{TASK_ID_EXAMPLE}}`).

## Schema

**Single source of truth for the schema = `docs/_harness/quality-gates.md` (§ handoff JSON schema).**
This README only describes the convention. Summary of required fields:

- `task_id`, `stage`, `agent`, `score`, `threshold`, `status`, `retry_count`
- `artifacts{}` — `files_written`/`files_modified` + `pytest_result{passed,failed,skipped}` + `coverage_pct` + `mypy_errors`
- `citations[]` — `{file, line, note}` (AUDIT verifies existence via grep)
- `findings[]` — `{severity, file, line, rule, fix_hint}`
- AUDIT entry: `hallucination_flags[]`, `permission_matrix_violations[]`, claimed-vs-actual re-verification table

## AUDIT Pass Condition

`score ≥ .harness.toml [gates].audit_threshold AND status == "pass"
AND hallucination_flags == [] AND permission_matrix_violations == []`

Any mismatch → `status: "fail"` + commit blocked (re-verified by the PreToolUse hook `scripts/harness_gate_check.sh`).
